1. Open a terminal window and type the following command:
```bash
rm -rf ~/.ssh/known_hosts
```
This will remove all the known hosts file from your local machine ssh client's configuration directory. 

Note: This command will only remove the known\_hosts file in your local machine. If you want to remove known\_hosts files from other machines or remote servers, please navigate to that directory and execute similar commands.

SSH（Secure Shell）是一个安全的网络协议，用于远程登录和管理服务器。它使用公私密钥加密技术来保证数据安全，而其中一部分功能就是记住已经连接过的主机信息，以便快速重新连接。

在Windows上，SSH客户端可以通过使用 PuTTY 工具来实现。PuTTY 工具维护了一个名为“ known\_hosts”文件的配置文件，用于记录已经建立的 SSH连接。当你想清除这个配置文件时，可以采取以下步骤：

1. 打开 Command Prompt（cmd）作为管理员权限。
2. 输入以下命令并按回车键：
```javascript
reg delete "HKEY_CURRENT_USER\Software\SimTelnet\SshKnownHosts" /v * /f
```
这会删除“ known\_hosts”配置文件中的所有信息。
3. 输入以下命令并按回车键：
```javascript
reg delete "HKEY_CURRENT_USER\Software\SimTelnet\SshHostKeys" /v * /f
```
这会删除“ SshHostKeys”配置文件中的所有信息。
4. 输入以下命令并按回车键：
```javascript
reg delete "HKEY_CURRENT_USER\Software\SimTelnet\SshHosts" /v * /f
```
这会删除“ known\_hosts”配置文件中的所有信息。
5. 输入以下命令并按回车键：
```javascript
reg delete "HKEY_CURRENT_USER\Software\SimTelnet\SshHostKeys" /v * /f
```
这会删除“ SshHostKeys”配置文件中的所有信息。
6. 关闭 Command Prompt（cmd）窗口。

完成此步骤后，PuTTY 工具就没有记录任何已知主机信息了。但是需要注意的是，如果你之前使用 PuTTY 连接过某个主机，那么这台主机的公私密钥就会被存储在 PuTTY 工具中的配置文件中，而不是“ known\_hosts”配置文件。如果希望清除 PuTTY 工具中的配置信息，可以尝试使用 PuTTY 工具内部的清理选项（通常可以在 PuTTY 工具的 “Preferences”（偏好）菜单中找到）。

```bash
ssh-keygen -R <IP_address>
```
上述命令中的`-R`选项用于指定要添加到所有已知主机中的IP地址。如果你得到了提示“mkstemp: No such file or directory”这是因为ssh-keygen工具需要使用临时文件来生成密钥对，但是由于某些原因无法在当前机器上创建临时文件。

可能的解决方案包括：

1. 运行命令时缺少权限。请以管理员身份重新运行命令。
2. 修改`SSH_PRIVATE_KEYFILE`环境变量，指定一个存在且可读写的文件作为密钥文件。例如：
```bash
export SSH_PRIVATE_KEYFILE=/path/to/your/keyfile.pem
ssh-keygen -R <IP_address>
```
3. 使用`--force-idempotency`选项，这可能会在某些情况下避免mkstemp的问题。例如：
```bash
ssh-keygen -R -f <IP_address>
```
4. 使用`--create-dirs-first`选项，如果当前目录没有必要的子目录，就会创建它们。例如：
```bash
ssh-keygen -R -C <IP_address> --create-dirs-first
```
5. 在生成密钥之前先检查当前机器是否缺少必要的文件，例如`~/.ssh/id_rsa`和`~/.ssh/known_hosts`等。如果任何一个文件不存在或者没有权限，请手动创建或改变权限，然后再重新运行命令。
